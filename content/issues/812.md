We're seeing occasional lost writes under load. Two workers that share a buffer can both observe `dirty == true`, both flush, and the second flush clobbers the first with a stale length. Racy repro: ~1 in 400.

Whoever fixes this should read the idempotency discussion in #869 first — the "duplicate under retry" hazard there has the same shape, and I don't want two half-fixes that fight each other. The flaky test in #840 is almost certainly surfacing this too.
