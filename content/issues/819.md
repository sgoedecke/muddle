`parseCSV()` blows up with `RangeError: Maximum call stack size exceeded` once the input crosses roughly 65,536 rows. The recursion in `consumeField()` isn't tail-called, so we blow the stack. Big customers hit this constantly on their nightly exports.

The obvious fix is to make the parser iterative, but note that changing field-splitting behaviour risks breaking the wire-format compatibility we promised in #903. Please coordinate.
