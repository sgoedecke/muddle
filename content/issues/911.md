Our CLI is a mess to script against. Some errors exit `0`, some print usage to stdout instead of stderr, and `--json` still emits human-readable log lines on stdout that corrupt the JSON. Anyone piping our output has to parse around our inconsistencies.

We should adopt a convention: machine output on stdout, diagnostics on stderr, non-zero exit on failure, and `--json` means *only* JSON. Note the CLI also prints secrets at `--verbose`, which is the same leak as #890. And the flags we're about to change interact with the deprecation plan in #925.
