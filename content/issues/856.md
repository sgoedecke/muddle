`formatDate()` interpolates `${year}-${month}-${day}` by hand and assumes the server timezone. For other timezones the date is off by one; for RTL locales the layout breaks. We should delegate to a locale-aware formatter and stop building date strings by concatenation.

This is user-visible and interacts with the accessibility work in #940 (screen readers announce the malformed dates oddly), and it's why exported timestamps don't round-trip in the parser work in #819. It's also blocking the onboarding docs in #918, which have wrong date examples.
