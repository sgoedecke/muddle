Our SCA scanner flagged `fast-parse@2.1.0` for a regular-expression denial of service (CVE pending). A crafted input causes catastrophic backtracking and pins a CPU core. We use it in exactly one place — the CSV fast-path — which is user-facing.

Short term: pin to `2.0.7` which predates the vulnerable regex. Long term: the iterative parser rewrite in #819 removes the dependency entirely, so let's make sure that lands.
